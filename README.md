# 🧪 Фабрика гипотез

Система генерации и приоритизации проверяемых инженерных гипотез по снижению потерь
цветных металлов (Ni/Cu) с хвостами обогатительных фабрик. Решение задачи 1 хакатона.

**Вход:** Excel-отчёт института по хвостам (гранулометрия + минералогия потерь) + цель + ограничения.
**Выход:** ранжированный список гипотез с механизмом, ссылками на источники, оценкой
рисков, ожидаемым эффектом в тоннах и дорожной картой проверки. Экспорт в DOCX/CSV/JSON.

## Архитектура

```
Excel хвостов ─► Парсер ─► Диагностический слой (правила, не LLM):
                            где потери (класс крупности × элемент)
                            в какой форме (раскрытые / сростки / неизвлекаемые)
                            тип проблемы (недоизмельчение / флотация / шламы)
                                  │
Учебники PDF ─► Чанки ─► Гибридный поиск (BM25 + эмбеддинги Yandex) ─┤
Схемы/регламенты (vision-описания) ──────────────────────────────────┤
Экспертные пары (4 объекта) ─► few-shot ─────────────────────────────┤
                                  ▼
                    YandexGPT: генерация гипотез (structured JSON)
                    + self-consistency (N сэмплов → сведение)
                                  ▼
            Ранжирование: прозрачная взвешенная сумма (веса настраиваются)
            + штраф за отсутствие grounding + сигнал новизны по эмбеддингам
                                  ▼
              Streamlit UI · DOCX-отчёт · CSV/JSON задачи · фидбэк-луп
```

Ключевые решения:
- **Диагностика — детерминированная.** LLM получает готовый диагноз («в классе +71
  теряется 2681 т Ni, всё извлекаемо, доминируют сростки → недоизмельчение»),
  а не сырую таблицу. Это устраняет арифметические галлюцинации и делает вывод
  воспроизводимым.
- **Anti-hallucination.** Гипотеза обязана ссылаться на номера реальных пассажей
  базы знаний; ссылки на несуществующие пассажи вычищаются, гипотеза без
  источников получает штраф −0.25 к скору и помечается в UI/отчёте.
- **Прозрачный скоринг.** Итог = Σ веса × критерий; полная раскладка по компонентам
  видна в UI и API. Никаких чёрных ящиков.
- **Обучение на фидбэке.** 👍/👎 эксперта обновляет веса критериев
  (exponentiated gradient) — без дообучения LLM.
- **Few-shot без утечки.** При генерации по объекту X его собственные экспертные
  гипотезы исключаются из примеров.

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env       # заполнить YC_API_KEY, YC_FOLDER_ID, DATA_DIR

# положить материалы хакатона (учебники + Пример 1..4) в DATA_DIR, затем:
cd backend && uvicorn app.main:app --reload --port 8000   # API
streamlit run ui/app.py                                    # UI (из корня)
```

Или docker: `docker compose up --build`.

При первом старте строится индекс базы знаний (~2 с, BM25). Эмбеддинги Yandex
опциональны: без них поиск чисто лексический; добавить —
см. `backend/app/knowledge_base.py::build_embeddings`.

## API

| Метод | Путь | Что делает |
|---|---|---|
| POST | `/hypotheses/generate` | Excel + цель + ограничения → ранжированные гипотезы |
| POST | `/hypotheses/{id}/rerank` | пересортировка с новыми весами (мгновенно) |
| GET | `/hypotheses/{id}` | детали прогона |
| POST | `/hypotheses/{id}/feedback` | 👍/👎 эксперта → обновление весов |
| POST | `/ingest` | добавить документ в базу знаний (PDF/MD/TXT) |
| POST | `/export/report` | DOCX-отчёт |
| GET | `/export/tasks?fmt=csv\|json` | задачи для Jira/YouTrack |
| GET | `/health` | статус + расход токенов |

## Метрики качества (методология)

- **Groundedness**: доля гипотез со ссылками на реальные пассажи (gate-метрика).
- **Сравнение с экспертами**: для 4 объектов есть гипотезы специалистов Компании —
  слепое сравнение top-N системы с ними (few-shot по объекту при этом исключается).
- **Латентность**: цель < 2 мин на прогон; расход токенов виден в `/health`.

## Ограничения и roadmap

- Учебник Лодейщикова — скан без текстового слоя; в прод — OCR (Yandex Vision).
- Схемы флотации внесены как текстовые описания (vision-предобработка на этапе
  сборки); в прод — автоматический vision-пайплайн для новых схем.
- Roadmap: граф знаний (Neo4j) с ABC-поиском непрямых связей; суррогатная модель
  состав → извлечение с байесовской оптимизацией; novelty-checker по патентным базам;
  Qdrant вместо in-process индекса при росте корпуса.
