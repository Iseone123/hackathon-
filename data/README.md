# Данные (локально, не в git)

Папка создаётся при работе с проектом. Исходники и артефакты **не коммитятся** — слишком тяжёлые для GitHub.

## Структура

```
data/
  Пример 1/              # КГМК: docx гипотез + xlsx хвостов
  Пример 2/              # НОФ вкр
  Пример 3/              # НОФ мед
  Пример 4/              # ТОФ
  Дополнительные материалы/   # учебники PDF
  Схемы флотации/
  Регламенты/
  processed/             # кэш ingest (JSON)
  hypotheses/            # сохранённые прогоны API
  batch_analysis/        # отчёты точности
  exports/               # PDF/DOCX отчёты
  uploads/               # загрузки через UI
```

## В git (лёгкий конфиг)

- `examples.yaml` — маршрутизация RAG по «Пример 1–4» и case-hints (обязателен для тестов и генерации).

Тесты с processed-данными используют `backend/tests/fixtures/processed/` — локальный `data/processed/` для pytest не нужен.

## Первый запуск

1. Скопируйте демо-материалы в соответствующие папки (или возьмите у команды).
2. Проиндексируйте через API или Streamlit: «Пример 1», «Дополнительные материалы».
3. `processed/`, `hypotheses/`, `exports/` появятся автоматически после ingest и генерации.
